# Codex 작업 체크리스트

이 문서는 Codex가 이 저장소에서 작업을 시작하기 전, 작업 중, 작업 완료 후 반드시 확인해야 하는 필수 체크리스트입니다. Codex는 모든 작업에서 `docs/codex_workflow.md`와 이 문서를 함께 읽고, 사용자가 승인한 범위 안에서만 변경해야 합니다.

## 1. 작업 시작 전 필수 체크리스트
- [ ] 현재 브랜치를 확인했는가?
- [ ] `docs/codex_workflow.md`를 읽었는가?
- [ ] `docs/codex_task_checklist.md`를 읽었는가?
- [ ] 사용자가 요청한 작업 범위를 확인했는가?
- [ ] 수정 가능한 파일/폴더 범위를 확인했는가?
- [ ] 수정하면 안 되는 파일/폴더를 확인했는가?
- [ ] 외부 API 호출이 금지되어 있는지 확인했는가?
- [ ] 실제 API key, token, password를 추가하지 않는지 확인했는가?
- [ ] 문서 내용은 한국어로 작성해야 한다는 규칙을 확인했는가?
- [ ] 파일명, 폴더명, 라벨명, 코드 식별자는 영어로 유지해야 한다는 규칙을 확인했는가?
- [ ] 기존 dataset label을 변경하지 않는지 확인했는가?
- [ ] 요청 범위를 벗어난 기능을 구현하지 않는지 확인했는가?

## 2. 작업 계획 체크리스트
- 이번 작업 목표:
- 수정 예정 파일:
- 생성 예정 파일:
- 수정하지 않을 파일:
- 예상 검증 방법:
- 완료 기준:

## 3. 작업 중 체크리스트
- [ ] 요청 범위를 벗어난 파일을 수정하지 않았는가?
- [ ] 문서 인코딩이 UTF-8로 유지되는가?
- [ ] 한국어 문서가 깨지지 않는가?
- [ ] 기존 데이터셋 컬럼과 라벨을 유지하는가?
- [ ] 불필요한 대규모 리팩터링을 하지 않았는가?
- [ ] 사용자 승인 없이 다음 단계 작업을 진행하지 않는가?

## 4. 작업 완료 후 체크리스트
- [ ] 변경 파일 목록을 요약했는가?
- [ ] 새로 생성한 파일을 요약했는가?
- [ ] 수정한 파일을 요약했는가?
- [ ] 실행한 검증 명령어를 기록했는가?
- [ ] 검증 결과를 요약했는가?
- [ ] 남은 문제나 주의사항을 적었는가?
- [ ] 다음 작업은 제안만 하고, 승인 없이 구현하지 않았는가?

## 5. Codex 응답 형식
Codex는 작업 완료 후 항상 다음 형식으로 응답해야 합니다.

```markdown
작업 전 확인:
- [x] ...

작업 계획:
- 목표:
- 수정 파일:
- 생성 파일:
- 검증 방법:

작업 결과:
- 추가:
- 수정:
- 삭제:

검증:
- 실행 명령어:
- 결과:

주의사항:
- ...

다음 제안:
- ...
```

## 6. 작업 요청 템플릿
아래 템플릿은 Codex에게 새 작업을 요청할 때 재사용합니다.

```markdown
Before making changes, read docs/codex_workflow.md and docs/codex_task_checklist.md.

Task:
[이번 작업 내용]

Scope:
[수정 가능한 파일/폴더]

Do not:
- Do not call external APIs.
- Do not add real API keys.
- Do not change dataset labels without approval.
- Do not modify unrelated files.
- Do not implement the next step without approval.

Completion criteria:
- 작업 전 체크리스트를 확인한다.
- 작업 계획을 먼저 요약한다.
- 작업을 수행한다.
- 작업 완료 후 변경 파일과 검증 결과를 요약한다.
- 다음 작업은 제안만 한다.
```
